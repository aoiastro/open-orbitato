# Open Orbitato 設計図

Open Orbitato（オープン・オービタト）は、LiteRT-LMで動作する、LangGraphベースの自作AIエージェント（OpenClaw風）です。
宇宙の軌道（Orbit）を周回するジャガイモ（Potato）のように、ローカル環境で自立的かつ軽快に動作することを目指します。

---

## 1. システム構成・アーキテクチャ

```mermaid
graph TD
    User([ユーザー / WebUI]) <--> WebServer[Webサーバー / API]
    WebServer <--> AgentEngine[Agent Engine (LangGraph)]
    
    subgraph AgentEngine [LangGraph エージェント]
        State[(Agent State)]
        NodeChat[対話ノード / LLM] --> NodeRouter{ルーター / ツール判定}
        NodeRouter -->|ツール実行| NodeTools[ツール実行ノード]
        NodeTools --> State
        State --> NodeChat
        NodeRouter -->|終了| NodeEnd[終了]
    end

    AgentEngine <--> LiteRTLM[LiteRT-LM Engine]
    LiteRTLM <--> Model[(models/gemma-4-E2B-it.litertlm)]
```

### 主要コンポーネント
1. **Frontend / WebUI**:
   - ユーザーとのチャットインターフェース。
   - エージェントの内部状態や思考プロセス、ツール実行履歴を可視化。
2. **Agent Engine (LangGraph)**:
   - エージェントのワークフロー制御。
   - LLMからの応答、ツール呼び出し（Tool Calling）のループ、状態（State）の管理。
3. **LLM Engine (LiteRT-LM)**:
   - ローカルに配置された軽量なLiteRTモデル（Gemmaなど）を用いて、高速かつプライベートに推論を実行。
4. **Tools**:
   - システム操作、ファイル操作、情報検索など、エージェントが実行可能なツール群。

---

## 2. 実装フェーズ（計画）

### フェーズ 1: LangGraphの導入とCUIプロトタイプ
- LangGraphとLiteRT-LMを組み合わせた最小構成のエージェントの構築。
- ツール呼び出し（Tool Calling）のサポート。
- CUI（ターミナル）上での対話ループの動作確認。

### フェーズ 2: WebUIとAPIサーバーの構築
- FastAPI等を用いたバックエンドAPIサーバーの構築（ストリーミング応答、状態取得API）。
- HTML/CSS/JS またはモダンなフロントエンドフレームワークを用いたWebUIの作成。

### フェーズ 3: ツールの拡充と自律性の向上
- ファイル操作、コマンド実行などの実用的なツール群の実装。
- エージェントの自律的な思考（ReAct / Plan-and-Solveなど）の導入。
